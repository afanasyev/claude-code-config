---
description: Push branch and create pull request
allowed-tools: Bash(git:*), Bash(gh:*)
---

**Pull Request Format:**

```
<type>(<scope>): <description>

## Summary
- Key change 1
- Key change 2

## Test plan
- [ ] Test step 1
- [ ] Test step 2
```

**Rules:**

1. Title: `<type>(<scope>): <description>`
    - Types: feat, fix, refactor, style, docs, chore
    - Lowercase, imperative mood, no period
2. Summary: 3-5 bullets on WHAT and WHY changed
3. Test plan: Practical checklist for testing

**Context:**

Current branch: !`git branch --show-current`
Uncommitted changes: !`git status --short 2>/dev/null | head -5`
Existing PR: !`gh pr view --json url,state --jq '"URL: \(.url) State: \(.state)"'`
Commits to include: !`git log main..HEAD --oneline`
Changed files: !`git diff main --stat`

**Task:**

1. If uncommitted changes exist, warn user and stop
2. If PR already exists, show URL and stop
3. Push current branch to origin with `-u` flag
4. Analyze ALL commits on this branch (from main)
5. Create PR to `main` using `gh pr create`
6. Return the PR URL

**Example:**

```bash
git push -u origin $(git branch --show-current)

gh pr create --base main --title "feat(scope): description" --body "$(cat <<'EOF'
## Summary
- Change 1
- Change 2

## Test plan
- [ ] Test 1
EOF
)"
```

**IMPORTANT**

- Check for uncommitted changes first
- Check if PR already exists before creating
- Push first, then create PR
- Include ALL commits in summary, not just the last one
- Return the PR URL when done
- NEVER include Claude, AI, or assistant references in PR title or body
- NEVER add "Generated by Claude" or similar footers
- NEVER include Co-Authored-By with Claude or Anthropic
